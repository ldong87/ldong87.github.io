  1. ๏ Promotional sales impact analysis for a retailer with 100 Billion$ annual sales (report)                           

Apr. - May. 2018  

   In this project, the impact of promotional sales event was analyzed by modeling the sales as target through exploratory data analysis and statistical modeling. The raw data includes 417k sales records of 610 different items from Jan 2014 - Jan 2017 and the corresponding daily price from its 80 major competitors. With the target sales being modeled as Gamma, Poisson and Tweedie distributions, respectively, Generalized linear model / Gamma regression and gradient boosting tree based models were evaluated using carefully engineered features. In the end, sales lift, which was defined as the sales difference between promotional items and similar regular items, was predicted as a deliverable for decision making. Sufficient exploratory data analysis was visualized with plotly, which can demonstrate my plotting skills.


  1. ๏ Toxic comment classification for Jigsaw (project repo)                                                                                                

Feb. - Mar. 2018  

   This was a Kaggle contest that tried to identify and classify toxic online comments. I won a silver medal, ranked top 5% out of 4551 teams. This was a multi-label classification problem and probability needed to be predicted to compute the ROC AUC as the evaluation metric. One comment can be classified into multiple classes. There were two levels of learners. The first level included GRU/LSTM, textCNN, Multi-column DNN, Deep Pyramids CNN, GRU/LSTM+CNN ,Capsule Net, gradient boosting, logistic regression, and naive Bayes with SVM. They took input from language data (~159k comments). The second level learners is a gradient boosting model. It took input from the output of the first level learners as meta-features. Different word embeddings were used. In addition, Monte Carlo simulation was adopted to find weights when stacking the first level results.


  1. ๏ Auto insurance claim prediction for Porto Seguro (project repo)                                                                                                

Nov. - Dec. 2017  

   This was a Kaggle contest that tried to predict if a driver will claim insurance next year or not.   1. ๏ Improving Zillow’s home value estimation algorithm - Zestimate (team Zidmie & Kostadinov & L) (project repo)                     

Aug. - Oct. 2017  

    This was a Kaggle contest that tried to help improve Zillow’s current home value estimation algorithm. Our team won a silver medal, ranked top 1% out of 3779 teams and qualified for the next round million dollar award. I created a meta-learning algorithm that makes use of most of popular machine learning algorithms. There were two levels of learners. The first level included gradient boosting, SVM, kNN, stochastic elastic net, fast linear online learning, genetic programming and neural networks. They took input from preprocessed structured training data (160k+ entries and each has ~50 features). The second level learners included a couple of gradient boosting algorithms. They took input from the training data as well as the output from the first level learners as meta-features. This meta-feature based algorithm can boost performance significantly. This pipeline was coded in Python (+1000 lines) and differs from existing stacking framework by considering the weak temporal pattern in the data. This pipeline uses machine learning libraries such as XGB, LightGB, CatBoost, Scikit Learn, Vowpal Wabbit, etc.
